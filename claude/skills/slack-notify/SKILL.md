# Slack Notify

## Overview

DM the user via the ai-agent-army Slack bot when a task completes, stalls, or needs input. One-shot wrapper around `chat.postMessage`. Credentials live in 1Password â€” never on disk.

**Agent-neutral:** identical behavior and invocation for Claude Code and Codex.

## When to Use

- User explicitly said "ping / Slack / message me when ..."
- Long-running task (â‰Ą 5 min) finishes while user is likely away
- Agent hits a blocker and needs user input to resume

## When NOT to Use

- Short interactive tasks â€” user is watching the terminal
- Progress chatter mid-task â€” noisy, trains the user to mute the bot
- No explicit notification request *and* the task is under a minute

## Usage

The script lives at a stable absolute path reachable from both agent runtimes:

```bash
NOTIFY=/Users/johnwu/code/ai-agent-army/claude/skills/slack-notify/notify.sh

# Task done
"$NOTIFY" --done "PR #42 merged, tests green"

# Blocked â€” needs user input
"$NOTIFY" --blocked "need DB credentials to run migration 0042"

# Neutral info
"$NOTIFY" --info "deploy started, ETA 4m"

# Raw message, no prefix
"$NOTIFY" "hey"
```

Exit `0` on success (prints `sent ts=...`). Non-zero with stderr on failure.

## Credentials

**Primary path (env vars, zero friction):**

`$SLACK_BOT_TOKEN` and `$SLACK_USER_ID` are injected at shell startup via
`~/.env-secrets.sh` (generated by `./secrets.sh` from
`dotfiles/env-secrets.sh.tmpl`). Every interactive shell has them in env, so
agents can DM the user with no `op` call and no Touch-ID prompt.

**Fallback (auto-used if env vars unset):**

`notify.sh` falls back to `op read` against:
- `op://ai-agent-army/Slack/Bot User Auth Token`
- `op://ai-agent-army/Slack/MemberID`

Useful for shells that haven't sourced `~/.env-secrets.sh` (fresh CI runners,
EC2 agents before bootstrap, stripped subshells). Override the 1Password
lookup path via:
- `SLACK_NOTIFY_OP_VAULT` (default `ai-agent-army`)
- `SLACK_NOTIFY_OP_ITEM`  (default `Slack`)

**Requires:** `jq` and `curl` on PATH. `op` is only required if the fallback
is hit â€” if `op read` fails with an auth error, tell the user to run
`op signin`; do not retry in a loop.

## Output Styling

Status flags inject Slack emoji codes so messages render with colored glyphs in the Slack client while staying plain-ASCII in the source:

| Flag | Prefix |
|---|---|
| `--done` | `:white_check_mark: Done: ` |
| `--blocked` | `:warning: Blocked: ` |
| `--info` | `:information_source: ` |
| *(none)* | no prefix |

## Common Mistakes

| Mistake | Fix |
|---|---|
| Sending a notify on every turn | Only on explicit request or a real end-of-task signal. |
| Dumping full logs into the message | One line. Link to PR, paste the key line, that's it. |
| `op` not signed in â†’ non-zero exit | Ask user to run `op signin`; do not loop. |
| Hardcoding the token anywhere | Always `op read` â€” tokens rotate, copies leak. |
| Using this as a progress feed | Use terminal / task tracking for progress. This is a *signal*, not a stream. |
